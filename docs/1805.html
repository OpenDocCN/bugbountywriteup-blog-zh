端口扫描

在我的端口扫描过程中，我首先使用 rustscan 来快速找出开放端口的数量…

…然后在那些开放的端口上开始详细的 Nmap 扫描。这样会节省很多时间。

网络侦察

所以我们只有一个开放的端口，即端口80。我们试着列举一下。我首先启动了一个 ffuf 扫描来检查我们是否有任何隐藏的目录。没有什么有用的东西。

现在我决定启动wfuzz来检查我们是否有可用的子域，这次我们有3个。

只需在/etc/hosts文件中再次添加所有这些子域。访问所有三个子域名会给你相同的页面。

通过我的一个脚本，我试图从页面中抓取所有的‘href’链接，我得到了Zabbix版本的文档。现在，这确保了我们的Zabbix版本大于5.0.x，如果你在谷歌搜索Zabbix 5.0.x漏洞，你会得到SQL注入5.0版本。

最后，在端口80上花了很多时间，我开始了UDP扫描，这给了我一个开放的端口，即623。

刚刚谷歌了"ASF-rmpc port 623 pentesting "我在 hacktricks 网站上找到了做这件事的方法。智能平台管理接口(IPMI)服务正在此端口上运行。

现在根据博客来查看ipmi版本，我们必须使用这样的IPMI辅助模块

我们有了IPMI 2.0版。现在使用第二个命令来转储任何哈希(如果存在的话)。

幸运的是，我们有一个可供一个用户使用的散列。使用hashcat来破解hash。

提示:- "-m 7300 "

反壳

登录后，我们有一个漂亮的仪表板。现在是时候了解Zabbix了。所以 Zabbix 基本上是一个企业级的开源分布式监控解决方案。我试着通过Zabbix dashboard搜索reverse shell，得到了一个由 IppSec 制作的视频，但是版本比较老。视频中的重要信息是Zabbix特工。因此，用C语言开发的本机 Zabbix代理 可以在各种支持的平台上运行，包括Linux、UNIX和Windows，并从设备收集CPU、内存、磁盘和网络接口使用情况等数据。阅读手册和漫游仪表板，我得到了Zabbix代理。

注意:- 我们将重点放在Zabbix代理上，因为它包含一个关键字段，该字段有一个选项“ system.run ”，用于在主机上执行指定的命令。

导航到配置→主机您将看到提到的 主机 。现在，主机有不同类型的标签，如触发器，项目，网页等。我们感兴趣的是 物品 标签。

点击物品→点击 创建物品

现在，通过从下拉列表中选择 system.run 选项，用 reverse shell 命令填充前三个字段，尤其是关键部分。现在最重要的部分，system.run键有两个选项命令和模式。第一部分填充了反向外壳，对于第二部分，我们有两个选项等待和不等待。我两种都试过，但是在等待模式下，连接不稳定。

现在启动一个NC监听器，点击“test”按钮→“get value ”,我们就有了一个shell。
