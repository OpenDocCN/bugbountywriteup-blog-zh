既然我们有“nullahm ”,让我们试着浏览一下https://nullahm.s3.amazonaws.com

好消息是名为“nullahm”的bucket存在，坏消息是我们被拒绝访问。

事情可能有两种方式，要么我们在这里做一些暴力，要么设置AWS CLI并尝试列出文件和文件夹。

挑战3！

它说“让我看看你的数据库技能”，这是与数据库有关的东西。嗯，哦，有道理，我们从第二次挑战中得到了DB证书。

为了确认这一点，让我们对托管CTF的主机运行一次快速NMAP扫描。

NMAP结果显示mysql服务正在主机上运行，并且我们有来自“hidden.jpeg”的DB凭证

让我们用下面的命令连接到MYSQL服务

有很多重复的行，有什么更快的方法呢？

我们来看看wordpress的“wp_comments”表的结构。

评论的主要内容存储在 comment_content 栏中。

但是现在呢？

你记得我们的标志格式是 nullctf{XXX}

让我们运行一个快速的like查询。

select * from WP _ comment where comment _ content like“% nullctf %”；

这是我们的第三面也是最后一面旗帜！
