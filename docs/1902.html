Dav 是TryHackMe中的一个简单机器，我们将在其中使用基本枚举，了解更多关于 WebDAV 的信息，以及如何探索它以获得对服务器的访问。

通过在浏览器中访问服务器，我们可以看到默认的Apache页面，所以这里没有什么要做的:

接下来，让我们使用 MASSCAN 查看哪些端口是打开的:

masscan -p- --rate 10000 10.10.45.24
  • -p-:枚举所有TCP端口，从1到65535
  • --rate:每秒向主机发送多少数据包

我更喜欢使用MASSCAN来枚举端口，因为它比nmap快得多，所以在发现端口后，我们可以使用nmap来单独检查这些端口中运行的服务。

从输出中，我们看到只有端口80 是打开的，所以现在我不会运行nmap，相反，让我们通过用 Gobuster 枚举目录来检查我们能找到什么:

gobuster dir -w /usr/share/wordlists/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt -t 50 -u 10.10.45.24
  • dir:告诉Gobuster枚举目录
  • -w:用于枚举主机的单词表，我通常使用 SecLists 中的这个
  • -t:要用多少根线
  • -u:目标网址

接近机器

从上面我们可以看到，Gobuster找到了一个有趣的目录:/webdav，所以我们在浏览器上查一下:

WebDAV意味着Web分布式创作和版本控制，并允许用户在服务器上创建、更改和移动文件，因此您可以将其视为类似FTP或SMB的东西。由于它与文件管理一起工作，我们可以很快假设我们将需要在不久的将来上传一个反向shell。

在/webdav目录中，我们看到我们需要凭证来访问该页面。因为我总是试图避免暴力攻击，所以我总是检查的第一件事是看我正在探索的任何东西是否有默认凭证来为我节省一些(通常是很多)时间。

对webdav default credentials的基本搜索将把我们带到这个页面，并给我们默认的凭证:wampp:xampp。

权限提升

在寻找权限提升向量时，我总是测试的第一件事是列出用户的权限，并检查当前用户是否可以作为root用户执行任何操作:

sudo -l
  • -l:列出用户的权限

嗯，cat正是我们需要阅读的那根旗帜:

sudo cat /root/root.txt
