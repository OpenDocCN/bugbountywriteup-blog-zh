## 负载箱

在本节中，我们将解释什么是 SQL 注入，描述一些常见的例子，解释如何发现和利用各种 SQL 注入漏洞，并总结如何防止 SQL 注入。

## 什么是 SQL 注入(SQLi)？

SQL 注入是一个 web 安全漏洞，使得攻击者能够干扰应用程序对其数据库的查询。它通常允许攻击者查看他们通常无法检索的数据。这可能包括属于其他用户的数据，或者应用程序本身能够访问的任何其他数据。在许多情况下，攻击者可以修改或删除这些数据，从而导致应用程序的内容或行为发生持久变化。

在某些情况下，攻击者可以升级 SQL 注入攻击来危害底层服务器或其他后端基础设施，或者执行拒绝服务攻击。

## SQL 注入类型:

*   **带内 SQLi(经典 SQLi) :** 带内 SQL 注入是最常见也最容易被利用的 SQL 注入攻击。当攻击者能够使用相同的通信信道来发起攻击和收集结果时，就会发生带内 SQL 注入。两种最常见的带内 SQL 注入类型是基于错误的 SQLi 和基于联合的 SQLi。
*   **基于错误的 SQL Li:**基于错误的 SQL Li 是一种带内 SQL 注入技术，它依靠数据库服务器抛出的错误消息来获取关于数据库结构的信息。在某些情况下，仅基于错误的 SQL 注入就足以让攻击者枚举整个数据库。
*   **基于联合的 SQLi :** 基于联合的 SQLi 是一种带内 SQL 注入技术，它利用联合 SQL 操作符将两个或多个 SELECT 语句的结果组合成一个结果，然后作为 HTTP 响应的一部分返回。
*   **推理 SQL Li(盲 SQL Li):**推理 SQL 注入与带内 SQL Li 不同，攻击者可能需要更长的时间来利用它，但是，它与任何其他形式的 SQL 注入一样危险。在推理式 SQLi 攻击中，实际上没有数据通过 web 应用程序传输，攻击者无法看到带内攻击的结果(这就是为什么这种攻击通常被称为“盲目 SQL 注入攻击”)。相反，攻击者能够通过发送有效负载、观察 web 应用程序的响应和数据库服务器的最终行为来重建数据库结构。两种类型的推理 SQL 注入是基于盲布尔的 SQLi 和基于盲时间的 SQLi。
*   **基于布尔(基于内容)的盲 SQL Li:**基于布尔的 SQL 注入是一种推理式 SQL 注入技术，它依赖于向数据库发送 SQL 查询，根据查询返回的结果是真还是假，强制应用程序返回不同的结果。根据结果，HTTP 响应中的内容会发生变化，或者保持不变。这使得攻击者能够推断所使用的有效负载返回的是真还是假，即使数据库中没有返回任何数据。
*   **基于时间的盲 SQL Li:**基于时间的 SQL 注入是一种推理式 SQL 注入技术，它依赖于向数据库发送 SQL 查询，强制数据库在响应前等待指定的时间(以秒为单位)。响应时间将向攻击者表明查询的结果是真还是假。根据结果，HTTP 响应将延迟返回，或者立即返回。这使得攻击者能够推断所使用的有效负载返回的是真还是假，即使数据库中没有返回任何数据。
*   **带外 SQL Li:**带外 SQL 注入并不常见，主要是因为它依赖于 web 应用程序所使用的数据库服务器上启用的特性。当攻击者无法使用同一通道发起攻击并收集结果时，就会发生带外 SQL 注入。带外技术为攻击者提供了基于时间的推理技术的替代方案，尤其是在服务器响应不太稳定的情况下(使得基于时间的推理攻击不可靠)。
*   **基于语音的 sql 注入:**这是一种 Sql 注入攻击方法，可应用于通过语音命令访问数据库的应用程序。攻击者可以通过发送带声音的 sql 查询从数据库中提取信息。
