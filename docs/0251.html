crackme0x00a

使用文件命令收集信息

$file crackme0x00a
  • 该文件为32位， LSB 可执行(最低有效字节)。
  • 这意味着文件是小端的。

使用 rabin2 工具收集信息。

$rabin2 -I crackme0x00a

使用字符串破解文件

$strings crackme00x0a

使用xxd破解文件

$xxd crackme0x00a

使用rabin2破解文件

使用ltracre破解文件

$ltrace ./crackme0x00a

ltrace基本上截取并打印程序执行的系统调用。

使用strace破解文件:(

strace ./crackme0x00a

strace用于跟踪系统调用。它不会跟踪像strncmp这样的字符串比较，因为它不需要任何系统调用。

使用radare2破解文件

$radare2 crackme0x00a
 [0x08048430]> aaa
 [0x08048430]> pdf @ main
  • aa:-全部分析。
  • aaa:-用更多信息分析所有内容。
  • pdf:-打印反汇编功能。

radare2给出了代码的视图。

  • 有一个*s2字符型指针，它指向一些字符串，即我们的输入(密码)。*s1是“g00dJ0B！”在下一步中，使用strcmp对两者进行比较。

使用gdb-peda破解文件

gdb-peda就像是gdb的一个插件，你可以从 GitHub 安装。

$gdb crackme0x00a
>gdb-peda$ disassemble main

反汇编主 将显示二进制的主功能。

在上有一个strcmp指令，所以让我们在这个位置设置断点并运行程序。


gdb-peda$ break *0x0804852a
gdb-peda$ run

输入密码来测试程序，在这种情况下，我输入 化名 作为密码。

《微服私访》是和《g00dj0B》相比较的！，gdb-peda也展示了一些猜测论点。
