使用Shodan访问Grofers Grafana实例

原文：https://infosecwriteups.com/accessing-grofers-grafana-instance-using-shodan-52c585ada797?source=collection_archive---------0-----------------------#2021-09-08

大家好…

过了很长时间，我在使用简单的 Shodan 搜索的 Grofers 中发现了一个有趣的bug。该漏洞可能允许攻击者访问 Grofers 的内部API监控仪表板。这是我第二次向 Grofers安全团队报告。

我给不知道 shodan 的小简介！

Shodan，互联网内所有端口的搜索引擎，可以帮助企业识别并锁定安全漏洞Shodan是互联网上一切的搜索引擎。

漏洞利用场景:

总是我的bug赏金之旅我会从shodan search或者 crt.sh (子域枚举)开始。最近我在使用crt.sh的 Dunzo 上发现一个bug

从简单的shodan呆子开始…

我发现了多个与 Grofers 域相关的主机。但我观察到一个有趣的宿主。立即打开那台主机，它的一个 Grafana实例 登录仪表板…

但是…我不知道用户名和密码…

输入默认用户名和密码，如 admin: admin。

但这里有趣的部分是…当我输入用户名和密码( admin: admin )时，登录页面重定向到新密码页面，并得到一个警告“已登录”。

然后再次输入密码为 admin。同时，弹出两个警报…

  • 无效或过期的重置密码
  • 用户密码已更改

密码更新了。我可以访问完整的Grafana实例。

然后我和POC一起创建了一份报告，并提交给了Grofers安全团队。
